# Contributing

Thank you for your interest in ArhamOS.

---

# Development Principles

- Preserve modularity.
- Avoid unnecessary abstractions.
- Prefer composition over duplication.
- Follow existing architectural patterns.
- Keep documentation synchronized with code.

---

# Workflow

Every contribution should follow:

Design

↓

Implement

↓

Test

↓

Document

↓

Review

↓

Commit

↓

Tag

---

# Coding Guidelines

- Keep functions focused.
- Avoid duplicated logic.
- Use shared utilities.
- Keep Skills independent of browser logic.
- Keep Tools deterministic.

---

# Pull Requests

Every pull request should include:

- Purpose
- Summary
- Testing performed
- Documentation updates

---

# Documentation

If architecture changes, update:

- ENGINEERING_SPEC.md
- STATE.md
- DECISIONS.md

before merging.

---

# Architecture

ENGINEERING_SPEC.md remains the single source of truth.

Avoid introducing breaking architectural changes without documenting the rationale.
